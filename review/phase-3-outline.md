## 页数建议

**默认：2.5–3 页 A4。**

理由：
- 15 年经验、Senior/Staff 级别、5 家公司、10+ 项目，2 页会过度压缩。
- 国内互联网招聘对 P7 / 技术专家 / 架构师 / 安全专家通常接受 3 页左右。
- 真正决定面试率的不是页数，而是**第一页是否吸引人**以及**每页阅读成本是否可控**。

### 三页布局

| 页码 | 内容 | 目标 |
|------|------|------|
| **Page 1** | Summary + Core Skills + 最近两家公司（闪捷 + 玩物得志） | 30 秒内让招聘经理知道「你是谁」 |
| **Page 2** | 闪捷最重要的 2–3 个项目 + 玩物得志核心项目 | 证明最近 5 年的深度与影响力 |
| **Page 3** | 蘑菇街及更早经历 + 教育 + CISSP + 开源/专利 | 证明成长轨迹与长期工程纪律 |

**关键原则**：
- 第一页信息密度最高，但要有留白。
- 每个 bullet 控制在 1–2 行。
- 关键量化结果加粗或独立成段。

### 信息预算

- **总字数**：中文约 2000–3000 字。
- **最近 5-10 年经历**：~65%
- **中期经历（蘑菇街）**：~20%
- **早期经历**：~10%
- **教育 / 证书 / 其他**：~5%

### 每家公司 Bullet 预算

| 优先级 | 每家公司 Bullet 数量建议 |
|--------|------------------------|
| P0（闪捷） | 每项目 2–4 个 bullet，全公司 10–15 个 bullet |
| P1（玩物得志） | 每项目 2–3 个 bullet，全公司 6–9 个 bullet |

| P3（早期） | 每段经历 2–3 个 bullet |
