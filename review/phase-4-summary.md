# Phase 4 — Resume A/B Summary 设计

## 设计原则

- 第一句直接建立专业身份。
- 不出现软技能或性格描述。
- 每句包含 ATS 关键词或能力信号。
- 体现 Career Arc：电信级工程纪律 → 高性能基础设施 → 企业安全 / AI 基础设施。
- 不写具体客户名，只写行业领域。

## Summary v4（根据中国技术总监阅读习惯重新设计）

### 设计原则

- 站在国内安全公司研发总监 / 技术总监视角。
- 第一句话建立「技术身份」：我主要是干什么的。
- 回答四个问题：我是谁、主要做什么、做到什么程度、技术边界是什么。
- 不使用营销语言：无「致力于」「赋能」「世界级」「行业领先」「职业主线」「演进」。
- 不堆砌关键词，形成清晰人设。
- 长度 70–120 字，最多 3–4 句话。
- 保持工程师风格，像资深工程师自己写的。
- 在具体性与概括性之间取平衡。

---

### Resume A — Application Security Engineer

#### v4 最终推荐版

```text
专注企业安全平台与应用安全产品研发，覆盖 AI 安全网关、API 安全网关、数据安全引擎、业务风控与内容安全。具备从 0 到 1 设计并落地安全产品的完整经验，熟悉 SDL、等保合规与安全审计。擅长用工程化方式解决安全风险，持有 CISSP。
```

**字数**：约 110 字，3 句。

#### 逐句解释

| 句子 | 功能 | 说明 |
|------|------|------|
| 第一句 | 建立技术身份 + 主要方向 | 「企业安全平台与应用安全产品研发」直接回答"我主要是干什么的"；后面列举具体产品方向，避免空泛。 |
| 第二句 | 做到什么程度 + 技术边界 | 「从 0 到 1 设计并落地」体现实战闭环；「SDL、等保合规、安全审计」明确安全工程边界，不是纯开发。 |
| 第三句 | 风格与资质 | 「用工程化方式解决安全风险」强化工程师人设；「CISSP」作为背书，但不喧宾夺主。 |

#### 删除/保留说明

- **删除**：
  - 「安全体系建设运营」→ 过于抽象，改为「应用安全产品研发」。
  - 「规模化安全运营」→ 偏运营/管理语言，删除。
  - 「服务金融、政务、互联网等行业」→ 下放到项目经历中体现，Summary 不需要承载。
  - 「在高速网关、数据安全与风险防控等方向有持续实践」→ 偏总结性，信息量低。
- **保留**：
  - AI 安全网关、API 安全网关、数据安全引擎、业务风控、内容安全 → 具体产品方向，构成清晰人设。
  - 0→1、SDL、等保合规、安全审计 → 证明能落地、懂安全工程。
  - CISSP → 安全资质背书。

#### 技术总监视角评价

20 秒内可建立的人设：**「这是一个做企业安全产品的人，懂开发也懂安全工程，有 0→1 落地经验，持有 CISSP。」** 关键词不堆砌，产品方向清晰，值得继续看项目。
