## 一、年度核心工作

### 1. 产品方向

- **深入一线理解市场需求**：前入一线与客户沟通，明确客户安全出发点与核心诉求
- **反馈产品问题**：聚焦稳定性、交互体验、功能缺陷
- **宣导产品价值**：从风险视角和安全体系建设视角宣导产品定位
- **输出产品需求**：风险规则、告警抑制、稳定性方案、交互体验优化、脏业务回退

**关键洞察**：API 安全不是强合规产品，面向市场解决实际安全问题。从 OWASP Top10 规则分类分级推演，消解一眼可见的负向体验（资产/告警过多、大量误判、系统卡顿、脏数据删不动）。

### 2. 项目交付

#### 3.1.3 版本（网关项目重启）

以招商、浙商大数据局需求为契机，将各项目支持分支收归主线：

- **代理能力**：复杂场景下通过响应头替换、JS 替换等实现**无损代理**
- **脱敏能力**：独立出脱敏服务，不同场景的脱敏能力大幅提升，灵活性大幅增加

#### 3.2.0 版本（重大版本升级）

| 维度 | 内容 |
|------|------|
| **新增** | 产品拆分、分布式部署、脱敏水印、告警展示 |
| **强化** | 黑白名单、访问控制、风险防护、交互体验 |
| **亮点** | 账号发现、水印形态多样性、小眼睛脱敏、规则灵活精确配置 |
| **工程** | 网关代码重写、部署升级、打通 JS 业务通道 |

**稳定性专项**：3.2.7.R100 → 3.2.7.R200，重点解决重大/基础负向体验，可运维、可回退。

### 3. 技术建设

- **Coding**：Nginx 模块重写，层次化/模块化的 Service 代码重构
- **安全体系**：系统学习甲方安全体系建设，通过 **CISSP** 认证
- **技术分享**：信息安全、客户沟通总结、效能提升

## 二、问题与挑战

1. **专注 vs 复合性**：如何平衡与外部资源之间的关系（公共模块、资产识别、敏感识别等）
2. **从 0→1 到 1→10**：
   - 开发边际产出降低，易产生懈怠效应
   - 代码熵增导致单位功能开发成本变高
   - 2B 系统"如何给飞机换引擎"
3. **价值审视**：对所有事情做充分的价值审视，一段时间内专注做好一件事

---

## 三、2024 年度规划

1. **持续打造业务亮点**
   - 结构化脱敏：深入业务，灵活自定义敏感数据，自适应规则学习
   - 非结构化脱敏：文件脱敏自研和产线结合，争取突破
   - 业务无损接入：网关 + 浏览器 JS 注入前后端配合，开拓加密、IAM 等场景

2. **扩展业务场景**：重点提升风险检测防护产品能力，支撑 API 加密、SDK 等新场景

3. **重点项目支持**：产品融合、华为 OEM 项目支持

4. **夯实工程基础**：持续推进代码优化和降熵，通过协议设计、工程结构提升稳定性和健壮性
